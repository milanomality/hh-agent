# HH-Agent

Личный ИИ-агент поиска работы на hh.ru: мониторит новые вакансии, оценивает их против
вашего резюме (LLM), пишет сопроводительные письма и показывает карточки в веб-интерфейсе
(доступен с телефона). Отклик совершаете вы сами по кнопке-ссылке — с 15.12.2025
соискательский API hh закрыт, автоотклики невозможны (и не эмулируются: это нарушение
правил площадки).

Архитектура и план — в [PLAN.md](PLAN.md).

## Быстрый старт

1. **Бэкенд.** Python 3.11+, зависимости:
   ```
   python -m venv .venv
   .venv\Scripts\python -m pip install -e .[dev]
   ```
2. **Фронтенд.** Node 18+; соберите SPA (бэкенд отдаёт готовый `frontend/dist`):
   ```
   cd frontend && npm ci && npm run build
   ```
3. Скопируйте `.env.example` → `.env` и заполните:
   - `WEB_PASSWORD` — пароль на вход; `WEB_SECRET` — секрет подписи куки
     (`python -c "import secrets;print(secrets.token_urlsafe(32))"`);
   - `HH_CLIENT_ID` / `HH_CLIENT_SECRET` — со страницы приложения на https://dev.hh.ru;
   - `LLM_API_KEY` — бесплатный ключ Groq (https://console.groq.com); альтернативы
     (OpenRouter / Gemini / локальная Ollama) — см. комментарии в `.env.example`.
     Платные модели по умолчанию не используются.
4. Положите резюме в `resume.md` (обычный текст или markdown: опыт, навыки, ожидания).
5. Запуск из корня проекта (uvicorn + поллер в одном процессе):
   ```
   .venv\Scripts\python -m hh_agent.main
   ```
   Откройте `http://localhost:8000`, войдите по `WEB_PASSWORD`.
   Для локального http:// поставьте `WEB_SECURE_COOKIE=false` в `.env`.
6. Во вкладке **Поиски** добавьте запрос (напр. «python разработчик»). Агент оценивает
   новые вакансии и кладёт карточки в **Вакансии** — с оценкой, разбором и готовым письмом.
   Откликаетесь по кнопке «Открыть на hh.ru», затем жмёте «✅ Откликнулся» — факт попадает
   в воронку (вкладка **Статус**).

Примечание: первый проход нового поиска может оценить до ~300 вакансий — это самый
затратный по токенам момент; дальше скорятся только новые. Веб-фид наполняется только
новыми проходами (карточки до перехода на веб в фид не попадут).

### Разработка фронтенда

Живой dev-сервер с проксированием `/api` на бэкенд (запущенный на :8000):

## Доступ с телефона / удалённо

hh.ru доступен из РФ напрямую и **не работает через зарубежный VPN/прокси** — клиент
намеренно ходит с `trust_env=False`. Значит процесс должен обращаться к hh.ru с
**российского IP**:

- **Рекомендуется:** домашняя всегда-включённая машина на российском канале + туннель
  (**Cloudflare Tunnel** `cloudflared`, ngrok или Tailscale Funnel). Через туннель идёт
  только входящий HTTPS-трафик, а запросы к hh.ru уходят с домашнего IP → hh доволен.
  За туннелем — `WEB_SECURE_COOKIE=true` (HTTPS есть).
- **Альтернатива:** **российский** VPS (Selectel / Timeweb / VDSina) + Caddy/nginx +
  Let's Encrypt.
- **Зарубежный клауд (AWS/Hetzner/DO) не подойдёт** — ломает доступ к hh.

LLM-нюанс: `trust_env=False` держит hh прямым даже при системном прокси, но SDK
LLM-провайдеров ходят по своему окружению. Выбирайте провайдера, доступного из РФ
(цепочка `providers.json`), либо прокси только для LLM-клиента.

## Docker (для VPS)

Multi-stage сборка (Node собирает `frontend/dist`, затем uvicorn):
```
docker build -t hh-agent .
mkdir -p data
docker run -p 8000:8000 --env-file .env \
  -e DB_PATH=/app/data/hh_agent.db \
  -e HH_TOKEN_CACHE_PATH=/app/data/.tokens.json \
  -v "$PWD/data:/app/data" \
  -v "$PWD/resume.md:/app/resume.md:ro" \
  -v "$PWD/providers.json:/app/providers.json:ro" \
  hh-agent
```
БД и кэш токена приложение создаёт само — поэтому монтируем каталог `data/` и указываем
пути в него (иначе `-v` на несуществующий файл создаст на хосте каталог, и sqlite упадёт).
`resume.md`/`providers.json` существуют заранее — монтируем файлами (read-only).
Держите один воркер uvicorn (одно sqlite-соединение и один поллер).
